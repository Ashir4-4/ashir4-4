# Run and deploy your AI Studio app

This contains everything you need to run your app locally.

View your app in AI Studio: https://ai.studio/apps/a548cafa-318c-4283-b075-8f4a379c6625

## Run Locally

**Prerequisites:**  Node.js


1. Install dependencies:
   `npm install`
2. Set the `GEMINI_API_KEY` in [.env.local](.env.local) to your Gemini API key
3. Run the app:
   `npm run dev`

## 🔑 سیستم مدیریت لایسنس و اشتراک کاربران

این پروژه یک سیستم کامل مدیریت لایسنس/اشتراک برای ربات تلگرام دارد
(`src/lib/ashir/license*.ts`) که مستقل از پنل کنترل معاملاتی ادمین کار
می‌کند و روی همان توکن تلگرام (`TELEGRAM_TOKEN`) سوار می‌شود.

متغیرهای محیطی قابل تنظیم در `.env`:

| متغیر | توضیح | پیش‌فرض |
|---|---|---|
| `LICENSE_ADMIN_IDS` | آیدی عددی تلگرام ادمین‌ها، جدا شده با کاما | همان `TELEGRAM_CHAT_ID` |
| `LICENSE_CARD_NUMBER` | شماره کارت بانکی برای پرداخت کارت‌به‌کارت | `0000-0000-0000-0000` |
| `LICENSE_CARD_HOLDER` | نام صاحب حساب | `نام صاحب حساب` |
| `LICENSE_PRICE_MONTHLY` | قیمت پکیج ماهانه (تومان) | `500000` |
| `LICENSE_PRICE_LIFETIME` | قیمت پکیج مادام‌العمر (تومان) | `4000000` |
| `WEB_ADMIN_USERNAME` | نام کاربری ورود ادمین به پنل وب | `admin` |
| `WEB_ADMIN_PASSWORD` | رمز عبور ورود ادمین به پنل وب | `admin123` ⚠️ حتماً عوض شود |
| `WEB_AUTH_SECRET` | رمز امضای توکن ورود پنل وب | باید در تولید تصادفی باشد |
| `EXCHANGE_KEY_ENCRYPTION_SECRET` | رمز اصلی رمزنگاری کلیدهای API صرافی کاربران | باید در تولید تصادفی و طولانی باشد ⚠️ |

### 💱 معامله‌ی خودکار روی حساب واقعی هر کاربر

هر کاربر می‌تواند از پنل وب (بخش «اتصال به صرافی XT» در حساب کاربری‌اش)
کلید API شخصی خودش را وارد کند. از آن پس، همان سیگنال‌های ورود/خروجی
که موتور اصلی اسکن تولید می‌کند (خرید/فروش/تارگت/حد ضرر)، به‌صورت
خودکار و **متناسب با موجودی خودِ همان کاربر** روی حساب واقعی خودش در
XT اجرا می‌شود — کاملاً مستقل از حساب دیگر کاربران و از حساب خود شما.

نکات امنیتی مهم:
- کلیدهای API کاربران هرگز متن ساده ذخیره نمی‌شوند؛ با AES-256-GCM و
  کلید `EXCHANGE_KEY_ENCRYPTION_SECRET` رمزنگاری می‌شوند.
- معامله‌ی خودکار برای هر کاربر پیش‌فرض **خاموش** است؛ کاربر باید صریحاً
  آن را از پنل روشن کند.
- به کاربران توصیه کنید کلید API را با دسترسی **«فقط معامله» (Trade
  only)** بسازند و هرگز دسترسی برداشت (Withdraw) را فعال نکنند.
- خطا یا قطعی در حساب یک کاربر (موجودی ناکافی، کلید نامعتبر و...)
  کاملاً ایزوله است و روی کاربران دیگر یا روی حساب اصلی تأثیری ندارد؛
  خود کاربر پیام خطا را در تلگرام دریافت می‌کند.
- ⚠️ این یک ربات معامله‌گر واقعی با پول واقعی است. قبل از فعال‌سازی
  گسترده، حتماً با مبالغ کوچک تست کنید.

مدت اشتراک تست (۷ روز) و ماهانه (۳۰ روز) و فاصله‌ی چک انقضا (۱۰ دقیقه)
در `src/lib/ashir/config.ts` قابل تغییرند.

کاربران عادی با زدن `/start` وارد جریان ثبت‌نام می‌شوند؛ ادمین (هر آیدی
داخل `LICENSE_ADMIN_IDS`) علاوه بر پنل معاملاتی معمول، دکمه‌ی «🛠 مدیریت
اشتراک‌ها» را هم در کیبورد خود می‌بیند. داده‌ها در فایل `license_users.json`
در ریشه‌ی پروژه ذخیره می‌شوند (مشابه الگوی `scanner_state.json` موجود).

### 🌐 پنل وب مدیریت/ورود کاربران

**کل سایت پشت یک صفحه‌ی ورود قرار دارد** — همان لحظه‌ای که کسی آدرس سایت
را باز می‌کند، اول صفحه‌ی لاگین را می‌بیند و بدون ورود موفق هیچ صفحه‌ای
(نه ترمینال معاملاتی و نه پنل کاربری) نمایش داده نمی‌شود:

- **ادمین** با `WEB_ADMIN_USERNAME` / `WEB_ADMIN_PASSWORD` (در `.env`) وارد
  می‌شود و مستقیماً همان ترمینال معاملاتی همیشگی را می‌بیند؛ با دکمه‌ی
  🛡 در هدر به «پنل مدیریت اشتراک‌ها» سوییچ می‌کند — جدول کامل کاربران با
  تایید/رد، بن/رفع‌بن، تمدید، افزودن دستی، و تعیین نام‌کاربری/رمز ورود
  برای هر کاربر.
- **کاربران عادی** فقط با نام‌کاربری/رمزی که ادمین برایشان از همان پنل
  تعریف کرده وارد می‌شوند و صفحه‌ی «حساب من» (وضعیت/نوع پکیج/تاریخ
  انقضا) را به‌صورت فقط-خواندنی می‌بینند.

اگر `WEB_ADMIN_PASSWORD` تنظیم نشود، در اولین اجرا مقدار پیش‌فرض
`admin` / `admin123` ساخته می‌شود — حتماً قبل از استفاده‌ی واقعی آن را
در `.env` عوض کنید. `WEB_AUTH_SECRET` هم باید در تولید به یک مقدار
تصادفی و طولانی تغییر کند (برای امضای توکن‌های ورود).
